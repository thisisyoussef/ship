# AI Workflow Index

These briefs organize the repo's AI-facing workflows under `.ai/workflows/` while mirroring the canonical guidance in `AGENTS.md` and `docs/`.

- `.ai/workflows/story-lookup.md`: find the next valid story and active execution contract
- `.ai/workflows/feature-development.md`: implement a story from preparation through validation
- `.ai/workflows/spec-driven-delivery.md`: use checked-in plans and specs without bypassing story execution
- `.ai/workflows/design-workflow.md`: route inspiration, canvas, visual evaluation, and implementation phases for design-heavy stories
- `.ai/workflows/visual-eval-loop.md`: run screenshot-driven critique and iteration when a design-heavy story needs it
- `.ai/workflows/parallel-flight.md`: coordinate shared harness work with `scripts/flight_slot.sh`
- `.ai/workflows/user-correction-triage.md`: track repeated correction loops with `scripts/triage_counter.sh`
- `.ai/workflows/story-handoff.md`: close a story with explicit validation, deployment status, and `What To Test`
- `.ai/workflows/git-finalization.md`: run default finalization and recover cleanly when it fails
