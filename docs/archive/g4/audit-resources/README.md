# Audit Resources

## Final deliverables kept at `docs/archive/g4/`

- `audit-report.md`
- `audit-report-visual.html`

## Supporting materials in this folder

- `audit-report-walkthrough.html`: annotated explanation of the audit decisions and tooling
- `codebase-orientation-reference.md`: earlier orientation notes used before the Phase 1 audit
- `ship-architecture-reference.html`: earlier architecture explainer

- `charts/`: generated SVG charts and a lightweight dashboard

## Suggested submission posture

Keep the two final deliverables in `docs/archive/g4/` as the primary submission artifacts. Treat everything in `audit-resources/` as supporting evidence and backup material.
