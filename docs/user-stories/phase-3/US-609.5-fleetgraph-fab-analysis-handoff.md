## Persona

**Engineer or PM** wants FleetGraph page analysis to open in the familiar floating chat surface so the analysis flow feels lighter and easier to continue.

## User Story

> As an engineer or PM, I want `Check this page` to open the FleetGraph FAB and run the current-page analysis there so I can inspect and continue the chat in the lighter analysis surface while keeping guided next-step actions on the entry card.

## Goal

Reintroduce the floating FleetGraph FAB as the canonical analysis/chat surface, while keeping the embedded entry card focused on guided-step preview and apply. `Check this page` should become a handoff into the FAB rather than rendering the analysis inline in the entry card.

## Scope

In scope:

1. Restore the FleetGraph FAB as the primary current-page analysis/chat surface on document pages.
2. Make `Check this page` on the entry card open the FAB and trigger the same current-page analysis there.
3. Preserve `Preview next step` and guided-step review/apply on the entry card.
4. Update proof-lane docs so they truthfully describe the split between FAB analysis and entry-card guided actions.

Out of scope:

1. New graph scenarios or prompt changes.
2. Reworking proactive findings behavior.
3. Broad visual redesign beyond the minimum needed to make the handoff feel coherent.

## UX Script

Happy path:

1. User opens a seeded FleetGraph document page.
2. User clicks `Check this page` in the entry card.
3. FleetGraph opens the floating FAB, runs page analysis there, and lets the user continue the chat.
4. If the user wants an action preview, they use `Preview next step` on the entry card instead of mixing that state into the FAB.

Error path:

1. User clicks `Check this page`.
2. FleetGraph either opens the FAB without analysis, or renders conflicting analysis state in both places.
3. The analysis surface feels duplicated and confusing.
